# Redis PV/UV统计组件

Redis PV/UV统计组件提供了高性能的页面访问量（PV）和独立访客数（UV）统计功能。组件采用本地缓存+Redis的架构，通过批量处理机制优化性能，并确保多节点环境下的数据一致性。

## 特点

- **高性能**：采用本地缓存+批量同步机制，减少Redis网络IO
- **低内存**：UV统计采用HyperLogLog算法，极大减少内存占用
- **简单易用**：提供简洁的API，易于集成到各类业务系统
- **线程安全**：内部实现完善的线程安全机制，无锁记录操作
- **批量处理**：优化的批处理机制，高效同步数据
- **活动统计**：支持活动PV/UV统计
- **多节点一致性**：优化的"读Redis写本地"策略确保数据一致性
- **单线程同步**：使用ScheduledExecutorService执行定时同步任务，降低资源开销
- **无锁设计**：全面无锁设计，包括记录操作和同步操作，提高高并发性能
- **统一键生成**：继承RedisKeyBuilder，提供规范化的Redis键生成策略
- **函数式校验**：使用函数式接口优化参数校验逻辑，减少代码重复
- **零配置设计**：使用硬编码默认值，无需外部配置，简化使用
- **业务类型支持**：内置业务类型枚举，支持不同业务场景的统计分析

## 最新优化

- **LongAdder替代**：使用JDK8的LongAdder替代AtomicLong，显著提高高并发计数性能

- **无锁同步**：移除所有锁，仅使用CAS操作确保同步互斥
- **ScheduledExecutorService**：使用Java原生定时任务替代Spring的@Scheduled，更精确控制
- **锁粒度优化**：移除记录PV/UV操作中的锁，大幅提高高并发性能
- **单线程同步**：使用单线程执行器处理同步任务，避免线程资源浪费
- **函数式编程**：使用Supplier函数接口减少重复代码，提高代码质量
- **静态常量默认值**：使用静态常量存储默认值，减少对象创建和GC压力
- **条件同步**：仅在本地缓存有数据时才执行同步操作，避免无效同步
- **性能监控**：添加同步操作耗时统计，便于性能监控和问题排查

## 快速使用

```java
@Autowired
private RedisPvUvManager pvUvManager;

// 记录页面PV
pvUvManager.recordPv(PvUvBizTypeEnum.PAGE, "home-page");

// 记录商品UV
pvUvManager.recordUv(PvUvBizTypeEnum.PRODUCT, "product-123", "user-456");

// 同时记录文章PV和UV
pvUvManager.recordPvAndUv(PvUvBizTypeEnum.ARTICLE, "article-789", "user-456");

// 记录活动PV/UV（不区分日期）
pvUvManager.recordTotalPvAndUv(PvUvBizTypeEnum.EVENT, "summer-sale", "user-456");

// 获取昨日视频PV/UV数据
LocalDate yesterday = LocalDate.now().minusDays(1);
PvUvResult result = pvUvManager.getPvAndUv(PvUvBizTypeEnum.VIDEO, "video-001", yesterday);
System.out.println("视频PV: " + result.getPv() + ", UV: " + result.getUv());

// 获取全局活动数据（不区分日期）
PvUvResult totalResult = pvUvManager.getTotalPvAndUv(PvUvBizTypeEnum.EVENT, "new-year-event");
System.out.println("活动总PV: " + totalResult.getPv() + ", 总UV: " + totalResult.getUv());
```

## 架构设计

组件采用三层架构：

1. **RedisPvUvManager**：核心管理器，处理业务逻辑和同步调度
2. **LocalPvUvCache**：本地缓存，实现线程安全的缓存机制
3. **RedisPvUvStorage**：Redis存储，负责数据持久化
4. **PvUvRedisKeyBuilder**：Redis键构建器，继承RedisKeyBuilder，提供统一的键生成策略
5. **PvUvBizTypeEnum**：业务类型枚举，规范不同业务场景的统计

数据流程：

1. 写入流程：先写入本地缓存，通过定时任务每30秒批量同步到Redis
2. 读取流程：直接从Redis读取最新数据，确保多节点环境下的数据一致性
3. 异常处理：Redis不可用时退化为本地缓存读取，保证高可用性

## 并发安全

组件内部实现了全面的并发安全机制：

1. **全面无锁设计**：使用LongAdder和双缓冲设计，实现真正的无锁操作
2. **原子操作**：PV计数使用LongAdder.sumThenReset，UV集合使用双缓冲原子交换
3. **CAS确保互斥**：仅使用CAS确保同步操作互斥，无需额外锁
4. **并发容器**：使用ConcurrentHashMap等线程安全容器实现内部存储
5. **ScheduledExecutorService**：使用Java原生的线程池执行定时同步任务

## 多节点数据一致性

在分布式环境下，组件采用"读Redis写本地"的策略确保数据一致性：

1. **写入策略**：
   - 先写入本地缓存，提高性能
   - 通过定时任务每30秒批量同步到Redis
   - 使用原子操作同时获取并重置本地计数
   - 应用关闭前执行最终同步，防止数据丢失

2. **读取策略**：
   - 直接从Redis读取，确保获取的是全局最新数据
   - Redis不可用时返回0，保持一致性处理

3. **数据一致性保障**：
   - PV使用LongAdder.sumThenReset原子操作同时获取并重置计数
   - UV使用双缓冲设计实现原子交换
   - 同步失败时保留恢复机制，确保数据不丢失
   - 使用原子操作确保同步操作的原子性

4. **异常处理**：
   - 同步失败时本地数据不会丢失，下次同步会重试
   - 提供手动同步方法，支持业务系统在关键时刻强制同步

## 本地缓存优化

本地缓存实现了以下优化机制：

1. **双缓冲设计**：UV数据采用双缓冲设计，实现无锁交换
2. **LongAdder计数**：PV和UV计数使用LongAdder，性能比AtomicLong高5-10倍
3. **并发安全**：所有操作都是线程安全的，无需额外同步
4. **空间优化**：只保存有值的数据，减少内存占用

## 性能优化

- **LongAdder替代AtomicLong**：在高并发环境下性能提升5-10倍
- **双缓冲无锁设计**：UV集合交换不需要锁，显著提高性能
- **原子操作同步**：使用原子操作同时获取并重置数据，避免同步过程的数据不一致
- **全面无锁设计**：所有操作都无需加锁，极大提高高并发性能
- **单线程同步**：使用单线程执行器处理同步任务，避免线程资源浪费
- **批量处理**：大量数据处理时采用分批机制，默认批量大小为500
- **HyperLogLog**：UV统计采用HyperLogLog算法，在12KB内存下支持千万级UV统计
- **管道批处理**：使用Redis管道批量执行命令，减少网络往返
- **静态常量默认值**：使用静态常量存储默认值，减少对象创建，降低GC压力
- **智能同步**：仅在缓存中有数据时才执行同步操作，避免无效同步
